## 🎯 Overview

<table>
<tr>
<td width="60%">

**Lemtik Security SOD** is the command interface of Africa's first civilian-grade **C4I (Command, Control, Communications, Computers & Intelligence)** platform — built specifically for the realities of Nigerian and African urban security environments.

When an emergency is reported, a security commander sitting at this dashboard sees the full picture in under **10 seconds**:

- 🧠 **AI-generated threat assessment** with confidence scoring
- 📍 **Closest available officers** ranked by proximity, capability, and ETA
- 🚗 **Vehicle availability** with real-time fuel status
- 🗺 **Optimal route** pre-calculated for each responder

One screen. Complete picture. Faster decisions. Lives protected.

### 🌐 Multi-Tenant Architecture

| Capability | Description |
|------------|-------------|
| 🏢 **Multi-Org** | Completely isolated data per organisation — estates, hotels, hospitals, banks, government agencies each see only their own data |
| 🔒 **Schema Isolation** | Dual-schema Supabase architecture (`sod` schema for operations, `services` schema for AI backends) with enforced role-based schema access |
| 👁 **Role Scoping** | Every query, every map pin, every alert — scoped to the authenticated user's organisation and location permissions |
| 📡 **Real-Time Scoping** | Socket.io rooms structured per organisation, per role, and per individual officer — no cross-org event leakage |

## 🚀 Getting Started

> **⚠️ Access Notice:** This is a proprietary, private repository. Access is restricted to authorised Lemtik Security team members only.

### Prerequisites

```bash
Node.js 20+ LTS
pnpm 9+          # pnpm-lock.yaml is the canonical lockfile
                  # do not use npm or yarn — version mismatch will occur
```

### Environment Setup

```bash
# 1. Clone the repository
git clone https://github.com/lemtikuniversalconcept/lemtiksecurity.git
cd lemtiksecurity/c4isod/lemtiksecurity-main

# 2. Install dependencies
pnpm install

# 3. Set up environment variables
cp .env.example .env.local
# Fill in your Supabase URL, anon key, Relationship API URL, and Mapbox token
```

### Required Environment Variables

```env
VITE_SUPABASE_URL=              # Your Supabase project URL
VITE_SUPABASE_ANON_KEY=         # Supabase anon key (public, scoped by RLS)
VITE_RELATIONSHIP_API_URL=      # http://localhost:3000 for local dev
VITE_MAPBOX_TOKEN=              # Mapbox public access token
```

### Local Development

```bash
# Start the dashboard (ensure Relationship API is running first)
pnpm dev

# Dashboard runs at: http://localhost:5173
```

### ⚡ Service Startup Order (Full Platform)

When running the full platform locally, services must start in this order:

```bash
# Terminal 1-6: Internal services (Python, each on a distinct port)
cd Internalservices/osint               && python app.py   # :8001
cd Internalservices/inventoryservices   && python app.py   # :8002
cd Internalservices/routecalculator     && python app.py   # :8003
cd Internalservices/proximity           && python app.py   # :8004
cd Internalservices/autonomouscontroller && python app.py  # :8005
cd Internalservices/masterai            && python main.py  # :8006

# Terminal 7: Relationship API (Node.js — start last among backends)
cd relationship_api && npm run build && npm start           # :3000

# Terminal 8: Dashboard
cd c4isod/lemtiksecurity-main && pnpm dev                  # :5173
```

### Build for Production

```bash
pnpm build       # Output to dist/
pnpm preview     # Preview production build locally
```
